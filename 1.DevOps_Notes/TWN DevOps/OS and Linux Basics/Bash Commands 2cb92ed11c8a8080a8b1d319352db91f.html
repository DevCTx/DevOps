    File tests:

    • -e : exists
    • -f : regular file
    • -d : directory
    • -r/-w/-x : readable/writable/executable
    • -s : not empty
    • -L : symbolic link
    • -nt/-ot : newer/older than

    String tests:

    • -z : empty string
    • -n : non-empty string
    • = (POSIX) or == (BASH) : equal
    • != : not equal
    • < / > : lexicographic order (in [[ ]])
    • =~ : regex match (in [[ ]] only)

    Numeric tests:

    • -eq : equal
    • -ne : not equal
    • -lt : less than (<)
    • -le : less or equal (≤)
    • -gt : greater than (>)
    • -ge : greater or equal (≥)

    Logic:

    • ! : negation
    • -a : AND (in [ ])
    • -o : OR (in [ ])
    • && and || : AND/OR (in [[ ]])

    Examples:

    if [ -f "file.txt" ] && [ -r "file.txt" ]; then ...
if [[ -f "file.txt" && -r "file.txt" ]]; then ...
if [[ $age -gt 18 ]]; then ...
if [[ $name =~ ^[A-Z] ]]; then ...
  • Bash Loops

    For loop:

    # Iterate over list
for item in file1 file2 file3; do
    echo $item
done

# Range (C-style)
for ((i=0; i<10; i++)); do
    echo $i
done

# Array
arr=(a b c)
for item in "${arr[@]}"; do
    echo $item
done

    While loop:

    while [ $i -lt 10 ]; do
    echo $i
    ((i++))
done

# Read file line by line
while read line; do
    echo $line
done < file.txt

    Until loop:

    until [ $i -ge 10 ]; do
    echo $i
    ((i++))
done

    Control:

    • break : exit loop
    • continue : skip to next iteration
  • Input parameters (arguments):

    Special variables:

    • $0 : script name
    • $1, $2, $3... : positional arguments
    • $# : number of arguments
    • $@ : all arguments (as list)
    • $* : all arguments (as string)
    • $? : return code of last command

    Examples:

    # script.sh arg1 arg2
echo "Script: $0"  #Script: ./script.sh
echo "First: $1"   #First: arg1
echo "All: $@"     #All: arg1 arg2
echo "Count: $#"   #Count: 2
    function check() {
    [[ -f "$1" ]] && return 0 || return 1
}

  • User / File / Output Readings

    Reading user input:

    # Basic read
read name
echo "Hello $name"

# With prompt
read -p "Enter name: " name

# Multiple variables
read first last
echo "$first $last"

# Read password (hidden)
read -s password

# With timeout
read -t 5 input  # 5 seconds

    Reading files:

    # Line by line
while read line; do
    echo $line
done < file.txt

# Or with IFS (field separator)
while IFS=',' read col1 col2 col3; do
    echo "$col1 - $col2"
done < data.csv

# Entire file into variable
content=$(<file.txt)
# or
content=$(cat file.txt)

    Reading command outputs :

    # With pipe (subshell)
count=0
ls -l | while read line; do
    ((count++))
done
echo $count #0 (lost variable!)

# With process substitution (current shell)
count=0
while read line; do
    ((count++))
done < <(ls -l)
echo $count # actual number of lines
    • <(command) : creates a temporary file with the output
    • < <(...) : redirects to stdin
  • Functions

    Basic syntax:

    # Two ways to declare
my_function() {
    echo "Hello $1"
}

function another_function {
    local result=$((${1} + ${2}))
    echo $result
}

# Calling
my_function "World"
another_function 5 3

    Key points:

    • No typing (like Python/JS, unlike C/C++/Java/C#)
    • Arguments via $1, $2, etc. (no explicit parameter list)
    • Return via echo or return (exit code 0-255 only)
    • Local variables with local

    Should use local for variable

    Global scope is no longer accessible after a local variable has shadowed it;

    therefore, use different names or save it beforehand:

    #!/bin/bash

counter=100

with_workaround() {
    local global_backup=$counter  # Save global BEFORE declaring local
    local counter=5
    
    echo "Local: $counter"
    echo "Global (shadowed): $counter"  # Still prints 5, not 100!
    echo "Global (saved): $global_backup"
}

with_workaround
