Package Manager
Get / Remove an official Package

APT Package Manager (dependencies shared between applications)

  • sudo apt search [package_name] = Search for a given package
  • sudo apt install [package_name] = Install a given package
  • sudo apt remove [package_name] = Remove installed package
  • sudo apt update = Updates the package index.

APT-GET Package Manager (Older than APT, but equivalent without graphical progression)

  • sudo apt-get install [package_name] = Install package with apt-get package manager

SNAP Package Manager (All in one file, so easiest to get but duplicate dependencies)

  • sudo snap install [package_name] = Install a given package
Get / Remove an non official Package

ADD-APT-REPOSITORY : Add a repository to APT sources into /etc/apt/source.list

if not in the official repositories yet

  • sudo apt-add-repository "deb http://repository.example.com/ubuntu stable main”
  • sudo apt update : need to update the APT index after !

ADD-APT-REPOSITORY —REMOVE : to remove it from the APT sources

  • sudo apt-add-repository --remove "deb http://repository.example.com/ubuntu stable main"
  • sudo apt update : need to update the APT index after !

PPA (Personal Package Archive) : used to obtain patches from developers before they are published in the official repository.

  • sudo apt-add-repository ppa:graphics-drivers/patch-driversX
  • sudo apt update
  • sudo apt install patch-driversX

  • Edit Mode: pressing ‘i’ allows to insert text

  • Press esc key = Switch to Command Mode
  • Type :wq = Write File to disk and quit Vim
  • Type :q! = Quit Vim without saving the changes
  • Type dd = Delete entire line
  • Type d10 = Delete next 10 lines
  • Type u = Undo
  • Type A = Jump to end of line and switch to insert mode
  • Type 0 = Jump to start of the line
  • Type $ = Jump to end of the line
  • Type 12G = Go to line 12
  • Type 16G = Go to line 16
  • Type /pattern = Search for pattern, e.g. /nginx
    • Type n = Jump to next match
    • Type N = Search in opposite direction
  • Type :%s/old/new = Replace 'old' with 'new' throughout the file

User and Group accounts
3 kind of users :
  • superuser (1 per system ) : no restricted permissions
  • user : each get a /home/user directory
  • service : relevant for linux server distributions like for Apach or MySql,

    and each service will have its own users

    • isolated from other users to launch their own applications in their scope
    • must not be run by root user !
Information are stored in 3 files

  • /etc/passwd : stores user account information (readable but only root-editable)

    alice:x:1001:1001:Alice Dupont:/home/alice:/bin/bash
    username : password : uid : gid : gecos : homedir : shell

    │ │ │ │ │ │ └─ user’s default shell

    │ │ │ │ │ └─ absolute user’s home path

    │ │ │ │ └─ additional user information

    │ │ │ └─ primary group stored into /etc/group

    │ │ └─ unique id of the user (0 kept for root)

    │ └─ x means it encrypted into /etc/shadow

    └─ Username used to log in

  • /etc/shadow : stores the password information (root-readable only)

    alice:$6$abcd...:19500:0:99999:7:14::

    username : hash : last : min : max : warn : inactive : expire : unused

    │ │ │ │ │ │ │ └─ account expiration

    │ │ │ │ │ │ └─ days before to disable

    │ │ │ │ │ └─ days before to warn user

    │ │ │ │ └─ maximum days before exipires

    │ │ │ └─ minimum days before changes

    │ │ └─ last change (in days since 01/01/1970)

    │ └─ hashed password with specifics or !/* if locked

    └─ username used to log in

  • /etc/group : stores the group information (root-readable only)

    sudo:x:27:alice,bob

    group : password : GID : members

    │ │ │ └─ comma-separated list of users in the group

    │ │ └─ numeric group ID

    │ └─ group password (usually 'x' or empty)

    └─ name of the group

Account Commands

User accounts:

  • sudo adduser [username] = Create a new user and a new group with username
  • sudo passwd [username] = Change password of a user
  • su - [username] = Login as username ('su' = short for substitute or switch user)
  • su - = Login as root

Group accounts :

  • sudo groupadd [groupname] = Create new group

2 types of commands :

  • adduser or addgroup : more interactive and automatic ways

    → to use when executing them manually

  • useradd or groupadd : to be more specific on information

    → better to use in shell or automated scripts

  • Idem for deluser vs userdel or delgroup vs groupdel

Add a group to a user:

  • sudo usermod -g devops tom = Assign 'devops' as the primary group for 'tom'
  • sudo usermod -G admin tom = Assign 'admin' as the secondary group and remove all other groups
  • sudo usermod -aG admin tom = Append 'admin' in secondary groups’ list

List the groups

  • groups = Display groups of the current user
  • groups [username] = Display the groups of a specific username

Remove a group from a user

  • sudo gpasswd -d username groupname : remove the group from the user’s goups
  • sudo deluser username groupname : another way

Delete a group from the list of groups

  • sudo groupdel groupname : removes the group from /etc/group.
Ownership and Permissions

Minus (-) removes the permission

  • sudo chmod g-w script.sh = remove 'write' permission for group

Plus (+) adds permission

  • sudo chmod g+x script.sh = Add 'execute' permission for group
  • sudo chmod u+x script.sh = Add 'execute' permission for user
  • sudo chmod o+x script.sh = Add 'execute' permission for other

Change multiple permissions for an owner

  • sudo chmod g=rwx script.sh = Assign 'read write execute' permissions to the group
  • sudo chmod g=r-- script.sh = Assign only 'read' permission to the group

Set permissions with digits

  • sudo chmod 777 script.sh = rwx permission for everyone
  • sudo chmod 740 script.sh = all for user, read for group, none for other

Network configuration
  • Ip Adresses and Masks

    Ipv4 address are composed with 4 octets

    ex : 192.168.0.1

    If we use a subnet mask of 255.255.255.0

    →all addresses from 192.168.0.0 to 192.168.0.255 are under the same LAN

    →means that 24 bits are fixed, so we can write it 192.168.0.0/24

    Network: 192.168.1.0/24 (255.255.255.0):

    • 256 total addresses (2^8)
    • 254 usable addresses (256 - 2)

      192.168.1.0 → Network address (not usable)
      192.168.1.255 → Broadcast (not usable)

    If we use a subnet mask of 255.255.0.0

    →all addresses from 192.168.0.0 to 192.168.255.255 are under the same LAN

    →means that 16 bits are fixed, so we can write it 192.168.0.0/16

    Network: 192.168.0.0/16 (255.255.0.0):

    • 65,536 total addresses (2^16)
    • 65,534 usable addresses (65,536 - 2)

      192.168.0.0 → Network address (not usable)
      192.168.255.255 → Broadcast (not usable)

SSH : Secure Shell

  • ssh username@SSHserver : Basic SSH connection
  • ssh -p 2222 username@SSHserver : Specify a custom port (default = 22)

Connection with a ssh key

  • ssh-keygen -t rsa : create a ssh key (private and public) in default location ~/.ssh
  • ls ~/.ssh/= Display contents of .ssh folder in /home, which has:
    • id_rsa = Private Key
    • id_rsa.pub = Public Key
  • ssh -i /path/to/private_key username@SSHserver : Use a private key

    example : ssh -i .ssh/id_rsa root@64.225.108.160 (file per default)

Two Files used by SSH:

  • ~/.ssh/known_hosts (On the client machine)

    Purpose:

    stores fingerprints of servers you’ve connected to before.

    → Client checks server

    How it works:

    1. First time you connect to a server, SSH shows the server’s fingerprint and asks if you trust it.
    1. If you accept, the fingerprint is stored in ~/.ssh/known_hosts.
    1. On future connections, SSH checks this file:
      • If the fingerprint matches → connection proceeds.
      • If it doesn’t → SSH warns you of a possible man-in-the-middle attack.
  • ~/.ssh/authorized_keys (On the server machine)

    Purpose:

    Lets the server authenticate the user based on their public key

    → Server checks client

    How it works:

    1. You generate an SSH key pair on your client
    1. Copy your public key to the server
      • This appends your public key to ~/.ssh/authorized_keys on the server.
    1. Next time you SSH in, the server checks if your private key matches one of the keys in authorized_keys.
      • If it matches → login allowed
      • If not → access denied

Execute remote command :

  • ssh username@SSHserver 'ls -l /var/log/’ : List remote log files
  • ssh username@SSHserver 'echo $SHELL’: Check what shell is running remotely

Debugging:

  • ssh -v username@SSHserver : Enable verbose mode (for debugging)
  • ssh -vvv username@SSHserver # even more verbose

SCP ; Secure Copy Protocol (SSH-based):

  • scp localfile.txt username@SSHserver:/remote/path/ : From local to remote
    scp -r /local/folder username@SSHserver:/remote/path/

    -r → recursive for directories.

  • scp username@SSHserver:/remote/file.txt /local/path/ :From remote to local

SFTP = Secure File Transfer Protocol (runs over SSH)

sftp username@SSHserver : Connect to the SFTP server

From local → remote :

put localfile.txt /remote/path/localfile.txt : Upload a single file

put -r /local/folder /remote/path/ : Upload a directory (recursive)

From remote → local

get /remote/file.txt /local/path/file.txt : Download a single file

get -r /remote/folder /local/path/ : Download a directory (recursive)

Useful SFTP commands

ls # list remote files
lls # list local files
cd # change remote directory
lcd # change local directory
pwd # remote working directory
lpwd # local working directory
exit

Use SSH tunnels / port forwarding:

  • ssh -L 8080:localhost:80 username@SSHserver : Local port forwarding
  • ssh -R 9090:localhost:3000 username@SSHserver : Remote port forwarding

Disconnect / exit :

  • Type exit or press Ctrl+D to close the SSH session.

KEYPOINT TO REMEMBER:

  • Config file for shortcuts: ~/.ssh/config
  • Always secure private keys (chmod 600 keyfile)
  • Combine p, i, and v for advanced connections
